设计上个前端blog网站的时候，希望实现几个功能

* 常规Blog，带有个性化样式
* 实现多语言，且不同语言显示样式
* 直接在网站上实现技术代码 demo

所以现在的 blog 网站的中英文代码不少都是单独写的。

时过境迁，我觉得 blog 有几个不舒服的点了：
* UI 样式我觉得过时了，可以更好看一点
* 中英文书写方式不够理想，需要我写 mdx，体验确实没有掘金网站这种写的方便**友好**
* 支持网页中demo，导致 blog 网站代码多了一些不该有的 sdk。
    * 且 sdk 版本变化，已经不能升级了，要升级就需要重构当时的 demo。可我不想这么做

我觉得，应该有个更好的 blog 网站了。需要**更好看、更方便、更干净**。

## 新主意
我决定不再维护英文。

最开始，写英文，希望自己能坚持表达的时候，知道英文是怎么说的。于是，写文章的时候，都要写一遍英文。慢慢的，大量使用翻译软件翻译，加上自己去检查。

AI 出现后，干脆就 AI 翻译，简单瞄一眼。

现在，这个我觉得还是让我写的有**负担**了。

之前或许对我的英文有帮助，但是英文的成长是多方面的，说不清是哪一件事情。现在这件事多是一件麻烦。中文的好些表达抽象，英文就要纳闷怎么去翻译，自己翻译一次就至少需要半小时。

blog，终归是为**自己记录，与小众分享**。这个小众，往往都是相知的自己人。


## 风格

风格上，还是需要更多的肃静。

少一点色彩，可以有一些俏皮的装饰。

去参考很多 blog 的巧思，装潢。

## 内容

不只是 blog、项目、about

还需要留存一些有用工具、网址、文章、推荐、游戏、音乐，写给自己笔记。也分享自己有用的东西，让 blog 网址更有价值。

## 图片与纪念
需要一个图库管理，与文字记录这种东西

## 文章系统设计

文章利用 github public 放置 md 文档。默认排序，加一个配置文件

每次更新就使用 github 提交 commit。
